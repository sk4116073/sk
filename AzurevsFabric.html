01 —

What are we comparing?

Organisations building modern data platforms on Azure typically assemble a stack of distinct managed services — each licensed, scaled, and billed independently. Microsoft Fabric, launched in 2023, consolidates this stack into a single SaaS platform with unified billing via Fabric Capacity Units (CUs).

The core question: should you stay on the Azure multi-service model, or migrate to Fabric? The answer depends on your organisation's size, complexity, existing investments, and growth trajectory — and this analysis covers all of it.

Microsoft Fabric — the unified model

Fabric bundles compute, storage, orchestration, BI, and governance into a single capacity-based purchase. You buy Fabric Capacity Units (CUs) — all workloads draw from that shared pool with no separate egress charges and no additional Power BI Premium license at F64+.

Migration path — small to large

Migration to Fabric is not all-or-nothing. Microsoft supports a phased approach that de-risks the transition while delivering early cost wins. For a medium org, expect 12–18 weeks from assessment to full decommission.

1

Assessment & inventory (weeks 1–2)

Catalogue all ADF pipelines, Databricks notebooks, Synapse pools, ADLS containers, and Power BI datasets. Tag by complexity and business criticality. Export Azure Cost Management reports to baseline current spend. Deliverable: migration backlog with effort estimates.

2 engineers × 2 weeksAzure Cost Management exportDeliverable: migration backlog
2

Provision Fabric + OneLake setup (weeks 2–4)

Start with F8/F16 for evaluation, scale to F32/F64 based on workload. Create OneLake workspace. Set up Fabric shortcuts pointing at existing ADLS — zero data movement, immediate read access. Configure Entra ID groups mirroring existing ADLS RBAC.

F32 capacity provisioningEntra ID workspace rolesROI starts: zero-copy ADLS accessDeliverable: live Fabric workspace
3

Migrate Power BI (weeks 3–5) — fastest ROI

Move all Power BI workspaces into Fabric. Enable Direct Lake mode on OneLake datasets — eliminates scheduled import refreshes entirely. Decommission Power BI Premium P1 ($4,995/month saved immediately). Users gain unlimited capacity access included in F64.

Direct Lake enablementSaves $4,995/month immediatelyRisk: semantic model reworkDeliverable: BI fully on Fabric
4

ADF pipelines → Fabric Data Factory (weeks 5–8)

Port ADF pipelines to Fabric Data Factory. Activity library is largely compatible — ~70% lift-and-shift. Remaining 30% (complex IR/custom activity) require refactoring. Run in parallel for 1 week before decommission to validate parity.

3 engineers × 3 weeks~70% lift-and-shiftSaves $2,200/month on ADFRisk: self-hosted IR equivalents
5

Synapse Analytics → Fabric Data Warehouse (weeks 7–10)

T-SQL compatible migration for Synapse Dedicated and Serverless Pool workloads. Run parallel query benchmarks for top 20 queries before cutover. Serverless Pool scans move to Fabric DW at zero per-TB-scanned cost within capacity.

2 engineers × 3 weeksT-SQL stored proc migrationSaves $6,500+/month on SynapseRisk: distribution/partition differences
6

Databricks — bridge & selective port (weeks 9–12)

Port notebook-based ETL to Fabric Spark (included in capacity). Databricks connects to OneLake as external Delta storage — no data copy needed. Retain Databricks only for ML model training where it adds unique value.

Notebook-to-Fabric-Spark portOneLake as Databricks external storageSaves $7,200–$9,000/monthRisk: Spark API compatibility
7

Consolidate, decommission & optimise (weeks 12–18)

Decommission all Azure services. Right-size Fabric capacity based on actual CU utilisation from the Capacity Metrics app. Apply Delta OPTIMIZE/VACUUM. Target: 30–60% total platform cost reduction fully realised.

Azure resource cleanupCU utilisation right-sizingDelta Lake optimisationDeliverable: Azure stack fully off

Migration cost estimate (medium org, 50 TB, 50–100 users): 12–18 weeks at 2–3 FTE = ~$65,000–$85,000 one-time. Recovered within 3–6 months through $10,000–$20,000/month reduction in platform costs.
